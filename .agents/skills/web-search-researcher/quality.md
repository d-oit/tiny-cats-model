# Quality Guidelines

## Source Priority

1. Official Documentation - Authoritative, maintained
2. Official Repositories - Code, issues, discussions
3. Expert Blogs - Practical experience
4. Stack Overflow - Real-world solutions
5. Academic Papers - Theoretical foundation

## Quality Indicators

Good sources have:
- Publication date (within 18 months)
- Author credentials/expertise
- Working code examples
- Maintenance/updates

## Red Flags

Avoid:
- No date or outdated information
- Generic content without specifics
- Contradicted by official docs
- AI-generated content farms

## Best Practices

- Include exact quotes with attribution
- Provide direct links
- Highlight conflicts and version-specific details
- Note gaps in available information
